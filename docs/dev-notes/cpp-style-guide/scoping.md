# 作用域

## 1. 命名空间

### 不具名命名空间（Unnamed Namespaces）

在 `.cc` 文件中，允许甚至提倡使用不具名命名空间，以避免运行时的命名冲突：

```cpp
namespace {                                    // .cc 文件中
// 命名空间的内容无需缩进
enum { UNUSED, EOF, ERROR };
bool AtEof() { return pos_ == EOF; }

}  // namespace
```

- 不能在 `.h` 文件中使用不具名命名空间
- 不具名命名空间结束时用注释 `// namespace` 标识

### 具名命名空间（Named Namespaces）

命名空间将除文件包含、全局标识的声明/定义以及类的前置声明外的整个源文件封装起来：

```cpp
// .h 文件
namespace mynamespace {

class MyClass {
 public:
  ...
  void Foo();
};

}  // namespace mynamespace

// .cc 文件
namespace mynamespace {

void MyClass::Foo() {
  ...
}

}  // namespace mynamespace
```

**规则：**
- 不要在命名空间 `std` 下声明任何内容
- 最好不要使用 `using` 指示符（`using namespace foo;`），以保证命名空间下的所有名称都可以正常使用
- 在 `.cc` 文件、`.h` 文件的函数/方法/类中，可以使用 `using`：

```cpp
// 允许：.cc 文件中；.h 文件中必须在函数/方法/类的内部使用
using ::foo::bar;
```

- 可以使用命名空间别名：

```cpp
namespace fbz = ::foo::bar::baz;
```

## 2. 嵌套类

当公开嵌套类作为接口的一部分时，虽然可以直接将它们保持在全局作用域中，但将嵌套类的声明置于命名空间中是更好的选择。

```cpp
class Foo {
 private:
  // Bar 是嵌套在 Foo 中的成员类
  class Bar {
    ...
  };
};
```

**结论：** 不要将嵌套类定义为 `public`，除非它们是接口的一部分。

## 3. 非成员函数、静态成员函数和全局函数

使用命名空间中的非成员函数或静态成员函数，尽量不要使用全局函数。

- 非成员函数不应依赖于外部变量，并尽量置于某个命名空间中
- 相比单纯为了封装若干不共享任何静态数据的静态成员函数而创建类，不如使用命名空间
- 如果你确实需要定义非成员函数，又只是在 `.cc` 文件中使用它，可使用不具名命名空间或 `static` 关联限定其作用域

## 4. 局部变量

将函数变量尽可能置于最小作用域内，在声明变量时将其初始化。

```cpp
int i = g();        // 好——初始化时声明
int i; i = f();     // 坏——初始化和声明分离
```

**效率注意：** 如果变量是一个对象，每次进入作用域都要调用其构造函数，每次退出作用域都要调用其析构函数。

```cpp
// 低效：构造和析构各 1000000 次
for (int i = 0; i < 1000000; ++i) {
  Foo f;
  f.DoSomething(i);
}

// 高效：构造和析构各 1 次
Foo f;
for (int i = 0; i < 1000000; ++i) {
  f.DoSomething(i);
}
```

## 5. 全局变量

- **class 类型的全局变量是被禁止的**（包括 STL 的 string、vector 等）
- 内建类型的全局变量是允许的
- **多线程代码中非常数全局变量也是被禁止的**
- **永远不要使用函数返回值初始化全局变量**
- 对于全局的字符串常量，使用 C 风格的字符串，而不是 STL 的字符串：

```cpp
const char kFrogSays[] = "ribbet";
```

大多数全局变量应该是类的静态数据成员，或者当其只在 `.cc` 文件中使用时，将其定义到不具名命名空间中，或者使用静态关联以限制变量的作用域。

> 静态成员变量视为全局变量，所以也不能是 class 类型！

## 小结

1. `.cc` 中的不具名命名空间可避免命名冲突、限定作用域，避免直接使用 `using` 提示符污染命名空间
2. 嵌套类符合局部使用原则，尽量不要 public
3. 尽量不用全局函数和全局变量，考虑作用域和命名空间限制
4. 多线程中的全局变量（含静态成员变量）不要使用 class 类型（含 STL 容器），避免不确定行为
