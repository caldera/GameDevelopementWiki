# Vampire Survivors — Roguelite 升级决策树深度分析

> **关联文档**：[Vampire Survivors 玩法拆解与设计分析](./vampire-survivors)

---

## 概述

Vampire Survivors 的升级系统虽然界面简单（3-4 个选项点一下），但其背后的机制极其精密。从权重系统、Luck 加成、拥有物品偏向到池子净化——这套升级系统决定了每一局的走向和节奏。

---

## 一、升级系统的整体流程

```
获得经验宝石     →  累计经验达到阈值   →  升级触发
                                               ↓
                                    游戏暂停（Time Stop）
                                               ↓
                                    从池中抽取 3-4 个选项
                                               ↓
                                    玩家选择 / 跳过 / 重选
                                               ↓
                                    获得物品 → 冷却重置 → 无敌帧
                                               ↓
                                    游戏继续
```

### 1.1 经验需求曲线

从 Wiki 提供的精确数据来看，经验需求不是简单的线性增长：

| 等级区间 | 每级增加量 | 累计到满级所需经验 |
|---------|----------|-----------------|
| Lv.1→2 | 5 | 5 |
| Lv.2→20 | +10/级 | 1,805 |
| Lv.20（特殊） | 额外 +600 XP（暂时 +100% Growth）| 2,600 |
| Lv.21→40 | +13/级 | 9,886.5 |
| Lv.40（特殊） | 额外 +2,400 XP（暂时 +100% Growth）| 12,800 |
| Lv.41→60 | +16/级 | 27,848 |

**关键设计点**：
- Lv.20 和 Lv.40 是**等级门槛**：需要额外巨量经验，但同时给 +100% Growth 加速
- 这相当于游戏在中期塞了一个"你要努把力才能突破"的瓶颈
- 突破后 Growth buff 让你快速升到下一阶段

### 1.2 升级时机与走位的关系

```
战斗状态 → 升级 → 暂停 → 选择 → 恢复 → 无敌帧
```

升级的选择游戏是暂停的，这天然给了玩家一个"安全思考"的时间。选择后的无敌帧保证了玩家不会被瞬间秒杀。

## 二、Rarity 权重系统

### 2.1 基础概率公式

```
P(某个武器被选中) = 该武器的 Rarity 值 / 所有可选武器的 Rarity 总和
```

所有的武器和被动道具都有一个 Rarity 值，决定了它出现在升级选项中的概率：

| Rarity | 约出现概率 | 代表武器 |
|--------|----------|---------|
| 1 | ~0.011% | 超稀有 |
| 40 | ~0.421% | 稀有 |
| 50-60 | ~0.5-0.6% | 较稀有 |
| 70-80 | ~0.7-0.8% | 普通稀有度 |
| 90-100 | ~0.9-1.05% | 常见 |

**重要数据**：所有 DLC 全齐的情况下，武器池总权重大约是 8,130，被动道具池是 1,370。

### 2.2 拥有物品偏向（Owned Item Bias）

这是升级系统最巧妙的设计之一。系统会**偏向**给你已经拥有的物品：

```
ownedChance = 1 + 0.3^(x-1) × totalLuck

其中 x = 2（当等级为偶数），x = 1（当等级为奇数）
```

**这意味着**：
- 如果你等级是偶数（x=2）：`ownedChance = 1 + 0.3 × totalLuck`
- 如果你等级是奇数（x=1）：`ownedChance = 1 + totalLuck`
- 奇数级时更容易抽到已有武器（更容易升级现有武器）

**系统会连续检查两次**，如果两次都成功，从已有武器池中抽取两个不同的武器作为选项。如果第二次和第一次抽到同一个，则丢弃第二个，从全武器池中重新抽取。

**设计意图**：
- 提高"升级已有武器"的概率 → 加速武器成型
- 偶数级/奇数级交替设计 → 自然的节奏：奇数级升级，偶数级补充新武器

### 2.3 第四个选项的生成

```
chanceFourth = 1 - (1 / totalLuck)
```

第四个选项纯靠 Luck 属性决定。Luck 越高，越容易看到 4 个选项而不是 3 个。更多选项 = 更高的选择自由度。

## 三、玩家的选择策略

### 3.1 三阶段决策模型

**阶段一：组建期（Lv.1-15 左右）**

```
目标：迅速建立基础战斗力
策略偏好：
- 优先拿武器（输出第一）
- 优先拿核心被动（如空书、Spellbinder）
- 尽量避免稀有不兼容的武器（避免卡槽）
```

**阶段二：成型期（Lv.15-40 左右）**

```
目标：补全进化路线，让武器全部满级
策略偏好：
- 拿缺失的被动道具（进化催化剂）
- 升级已有的武器（触发进化）
- 忍痛放弃不兼容的选项（使用 Banish）
```

**阶段三：极限期（Lv.40+ / Limit Break）**

```
目标：拓展已有武器的极限属性
策略偏好：
- 优先提升核心武器的关键属性
- 使用 Skip 跳过不需要的升级
- 追求超高 Amount/Area/Might
```

### 3.2 Banish / Reroll / Skip 的决策价值

玩家可以通过 PowerUp 系统获得三种操作能力：

| 操作 | 效果 | 最佳使用场景 |
|------|------|------------|
| Skip | 跳过一次升级（保留经验） | 三个选项都不想要时 |
| Reroll | 刷新三个选项 | 组要期缺关键武器时 |
| Banish | 永久移除一个选项 | 确定某个武器/被动不会用 |

**Banish 的战略意义**：因为已 Banish 的武器不会再出现，Banish 可以显著缩小池子，提高你想要的武器的出现概率。这是一种"用一次性的代价换取后续概率优化"的设计。

### 3.3 满级池子净化

```
武器满级后 → 不再出现在升级选项池中
```

**这个设计有多重要**：

```
假设你已经有 4 个武器满级：
- 这 4 个武器不再出现在池中
- 剩下 2 个武器 + 6 个被动 = 总抽取范围缩小
- 你更大概率抽到自己还想要的
```

这是 VS 隐性引导玩家"先专注几个武器满级"的设计——满级的武器越多，池子越干净，后面需要的越容易抽到。

### 3.4 满配后的选择

当 6+6 全部满级，且没有可进化的武器时：

```
选项会变成：
1. 金币 (Gold Coin) ≥ 100
2. 鸡腿 (Floor Chicken，如果血量不满)
3. Limit Break：武器属性升级（如果开启）
```

"金币或鸡腿"作为保底选项 → 满配后玩家仍然有升级的动力。

## 四、Luck 属性的深层影响

Luck 在升级系统中影响三个维度：

| 维度 | 计算公式 | 影响 |
|------|---------|------|
| 拥有物品偏向 | `1 + 0.3^(x-1) × totalLuck` | 越高越容易升级已有武器 |
| 第四个选项 | `1 - 1/totalLuck` | 越高越容易有更多选择 |
| 宝箱等级 | 乘算宝箱的掉落概率 | 越高越容易出高级宝箱 |
| 暴击率 | 基础暴击 × totalLuck | 越高暴击概率越大 |

**设计启示**：Luck 不是一个单一的"掉宝率"属性，而是一个**影响决策链的多维属性**。它同时影响：
- 你能看到多少选项（第四个选项）
- 你是否能升级已有武器（拥有物品偏向）
- 你宝箱能开出多好的东西（宝箱等级）

## 五、决策树的复杂性与重量

### 5.1 决策密度

在一局约 30 分钟的游戏中，玩家大约会升级 50-80 次：

```
升级次数 ~ 50-80 次
每次 3-4 个选项
→ 每局约 150-320 个"要不要选"的微决策
→ 约每 22 秒一次决策
```

对比传统 Roguelike（如 Slay the Spire）：一局约 1 小时，约 30-40 次战斗后选卡，每次选卡的决策重量更大。VS 的决策更轻量但更频繁。

### 5.2 决策重量

虽然每次升级的决策只有"点一下"那么轻，但每个决策都有后续影响：

- 拿了这把刀 → 占了一个武器槽 → 放弃了另一个武器
- 选了 Attractorb → 未来可能用不上它进化 → 浪费了一个被动槽
- 没拿魔法杖 → 后期刷出来也可能因为槽满了而拿不到

**这是"路径依赖"的决策树**：每一个选择都在缩小后续的选择空间。

### 5.3 是否有"最优解"？

严格来说，每局没有确定的最优解，因为：
1. 随机性导致你无法保证能拿到特定组合
2. 不同的角色/地图/模式需要的配装不同
3. Luck 属性在升级系统中的多维度影响让概率变得复杂

但有些**高概率好用的策略**是存在的，有经验的玩家会追求特定的武器/被动组合，同时保留灵活调整的能力。

## 六、Key Takeaways

1. **权重系统制造稀缺感**：低 Rarity 武器出现概率只有不到 0.1%，每次看到都像是在"中奖"
2. **拥有物品偏向 = 隐性引导**：系统偏向给你已有的武器 → 鼓励专注升级而不是分散投资
3. **池子净化是反直觉的聪明设计**：满级武器不再出现，让后期选择更精准
4. **所有属性都连接到了升级系统**：Luck 影响选项数量和偏向，Growth 影响升级速度
5. **决策快但重量高**：每次选择都有长远影响，但操作只需要点一下
