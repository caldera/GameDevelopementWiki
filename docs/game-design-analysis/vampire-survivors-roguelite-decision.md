# Vampire Survivors — Roguelite 升级决策树深入分析

> **关联文档**：[Vampire Survivors 玩法拆解与设计分析](./vampire-survivors)

---

## 概述

Vampire Survivors 的升级系统看似简单（每次 3-4 个选项点一下），但背后有一套精密的概率引擎。从权重系统、Luck 的多维度影响、拥有物品偏向到池子净化——这套系统决定了每一局的走向和节奏，是游戏"看起来随机、玩起来可控"的秘密。

本文所有公式和数据均来自 Wiki 的 `Level up` 和 `Luck` 页面。

---

## 一、升级系统的整体流程

### 1.1 完整事件序列

```
收集经验宝石 → 经验条满 → 升级触发
                                   ↓
                     经验宝石从屏幕上方落下的动画
                                   ↓
                     游戏完全暂停（Time Stop）
                                   ↓
                     从物品池中抽取 3 或 4 个选项
                                   ↓
                     ↓ 显示选项（稀有/常见武器名+图标）
                                   ↓
                     玩家操作：
                       ├─ 选一个武器/被动    → 添加到背包或升级
                       ├─ Skip（跳过，保留经验）
                       ├─ Reroll（刷新 3 个）
                       └─ Banish（永久移除一个选项）
                                   ↓
                     选择后：游戏恢复
                                   ↓
                     角色获得短暂无敌帧
                                   ↓
                     如果选的是已有武器 → 立即重置冷却并发射一次
                                   ↓
                     继续战斗
```

### 1.2 经验需求曲线（精确数据）

Wiki 页面中内置了一个经验计算器，以下是精确到每一级的数据：

| 等级 | 累计 XP | 段差特征 |
|------|--------|---------|
| Lv.1→2 | 5 | 最少的起步 |
| Lv.2→20 | 每级 +10 | 线性增长，Lv.20累计 1,805 |
| **Lv.20→21** | **+600** | **大幅跃升，同时给 +100% Growth** |
| Lv.21→40 | 每级 +13 | 新的线性段，Lv.40累计 9,886.5 |
| **Lv.40→41** | **+2,400** | **更大跃升，同时给 +100% Growth** |
| Lv.41→60 | 每级 +16 | 最终段，Lv.60累计 27,848 |

**Lv.20 和 Lv.40 的 Growth 爆发机制**：

> "At levels 20 and 40 an additional amount of XP—600 and 2400 respectively—is required to level up to the next level. However, at these levels the player also gains +100% Growth, increasing their experience gain, until they reach the next level."

```
Lv.19 → 20：需要 185 XP（正常）
Lv.20 → 21：需要 600 XP（异常高）  
           但这段期间 Growth × 2 → 每个经验宝石给你双倍经验
           所以虽然需求高了，但因为 Growth 翻倍，实际打怪数量≈ 300 的体验

Lv.39 → 40：需要 498.5 XP（正常）
Lv.40 → 41：需要 2,400 XP（极高）
           同样 Growth × 2 → 实际打怪数量≈ 1,200 的体验
```

**为什么设计这两个门槛**：
- 防止玩家等级增长过快（线性增长太容易预测）
- 制造两个"成长瓶颈"，考验玩家在中期是否站稳了
- 瓶颈后的 Growth 爆发让玩家感受到"突破"的快感
- 这两个门槛恰好对应游戏的前期→中期（Lv.20）和中期→后期（Lv.40）的过渡

### 1.3 Growth 属性的作用公式

```
实际获得经验 = 宝石基础值 × (1 + Growth%)

Growth 来源：
  PowerUp（金币升级）   max +15%
  Crown（被动道具）     max +40%
  角色加成              0%~+50%
  Wicked Season Arcana  周期性 +100%
  Hurry Mode            +25%
  Lv.20 门槛爆发        临时 +100%
  Lv.40 门槛爆发        临时 +100%
```

Growth 是加速成长的核心属性，但选 Crown 要牺牲一个被动槽位（且 Crown 在大多数进化中**不是催化剂**），所以选 Growth 意味着"我要快速成型"的战略取舍。

### 1.4 升级暂停的完整保护机制

```
1. 经验条满 → 升级动画触发
2. 游戏完全暂停（Time Stop）→ 所有敌人冻结
3. 界面显示 3-4 个选项
4. 玩家选择或 Skip/Reroll/Banish
5. 游戏恢复
6. 角色获得 0.5-1.0 秒无敌帧
7. 如果选的是已有武器 → 冷却立即重置 + 触发一次攻击
```

**升级暂停的设计价值**：
- 如果升级不暂停，玩家需要"边逃命边选技能" → 选错率高、体验差
- 暂停把"选技能"变成安全的决策时刻
- 无敌帧防止"刚点完就被面前的怪秒了"的极端挫败
- 冷却重置 + 立即攻击 → 恢复后不是输出空窗，而是输出爆发

## 二、Rarity 权重系统

### 2.1 基础概率公式（来自 Wiki）

```
P(某个物品被选中) = 该物品的 Rarity 值 / 所有可选物品的 Rarity 总和
```

```
Wiki 原文：
"Each item in the item pool has weight that is specified by their rarity value. 
The weight of the whole item pool is the sum of the rarity values of eligible items."
```

### 2.2 武器和被动道具的权重池

Wiki 给出了精确的池子权重总数据：

> "Currently, the total weight of the weapon pool with all DLCs is **8,130** and the passive item pool with all DLCs is **1,370** (total **9,500**)."

这意味着每次升级选项中的**单个槽位**概率分布如下：

| 武器池 | 权重 | 分摊方式 |
|--------|------|---------|
| 武器 (54+ 种) | 8,130 | 按 rarity 分配 |
| 被动道具 (22+ 种) | 1,370 | 按 rarity 分配 |
| 总计 | 9,500 | |

**关键结论**：武器 vs 被动道具的出现概率 ≈ 8,130 : 1,370 ≈ **86% : 14%**

也就是说，**绝大多数升级选项是武器**，被动道具出现概率很低。这是为什么玩家常常"武器的都六格满了，核心被动还没凑齐"。

### 2.3 单个 Rarity 值的精确概率

Wiki 提供了一个精确的概率表：

| Rarity | 约出现概率 | 每 100 次升级中看到的次数 |
|--------|----------|------------------------|
| 1 | **0.011%** | 0.011 次（几乎看不到） |
| 40 | 0.421% | 0.421 次（平均 237 次升级看到一次） |
| 50 | 0.526% | 约 0.5 次（190 次升级看到一次） |
| 60 | 0.632% | 约 0.6 次（158 次升级看到一次） |
| 70 | 0.737% | 约 0.7 次（135 次升级看到一次） |
| 80 | 0.842% | 约 0.8 次（119 次升级看到一次） |
| 90 | 0.947% | 约 1 次（106 次升级看到一次） |
| 100 | **1.053%** | 约 1 次（95 次升级看到一次） |

```
Rarity 1 的武器（超稀有）出现概率 ≈ 0.011%
这意味着一局 60-80 次升级中，看到它的概率 < 1%
你需要打 10+ 局才有可能看到一次
```

### 2.4 Rarity 值的分配实例

| 武器 | Rarity | 稀有度 |
|------|--------|--------|
| Whip | 100 | 常见（新手默认就带着） |
| Magic Wand | 100 | 常见 |
| Peachone / Ebony Wings | 100 | 常见（融合组件） |
| Song of Mana | 100 | 常见 |
| Pentagram | 80 → 10（Lv.2+） | 特殊（等级越高越不出） |
| Vento Sacro | 50 | 较稀有 |
| Bone | 50 | 较稀有 |
| Flames of Misspell | 70 | 中等 |
| Victory Sword | **1** | **超稀有** |
| Sole Solution | 稀有 | 进化后更难出 |

**Victory Sword（Rarity 1）**：
- 出现概率仅 0.011%
- 这是游戏中最稀有的武器之一
- 一局 60 次升级中看到它的概率不到 1%
- 一旦拿到 → 非常强力的剑气武器

### 2.5 Pentagram 的特殊 Rarity 递减

Pentagram 是唯一一个 **Rarity 会随等级下降**的武器：

```
Pentagram Lv.1:  Rarity = 80
Pentagram Lv.2+: Rarity = 10
```

**设计意图**：升了级的 Pentagram 不会再频繁出现（因为已经不需要了），避免浪费升级槽位。

## 三、玩家的选择策略

### 3.1 决策分层模型

VS 的升级决策可以按"决策的紧迫性"分层：

| 层 | 决策类型 | 时间窗口 | 例子 |
|----|---------|---------|------|
| 微决策 | 当前升级选什么 | ~5秒 | "拿魔法杖还是空书？" |
| 中决策 | 当前阶段配装方向 | 5-10 分钟 | "走冷却流还是 AOE 流？" |
| 宏决策 | 本局最终的进化路线 | 一局规划 | "凑 Infinite Corridor 还是走圣水？" |

### 3.2 Banish / Reroll / Skip 的战略价值

这三个操作来源于 PowerUp 系统，可以在升级时使用：

| 操作 | 说明 | 最佳使用时机 |
|------|------|------------|
| **Skip** | 跳过本次升级，保留经验 | 三个选项都不想要 → 等下一次升级赌更好选项 |
| **Reroll** | 刷新三个选项（消耗一次重roll次数） | 三个都不好但有"还差一个就能进化"的需求 |
| **Banish** | 永久标记一个物品为"不会出现" | 确定这把不需要某个武器/被动 → 缩小池子 |

**Banish 的数学价值**：

```
假设你已拿到 5 个武器，还差 1 个武器槽位：
共有 50 种武器在池中（没有 Banish 任何武器）
第 6 个武器的出现概率：每个选项 1/50 ≈ 2%

但如果你 Banish 了 20 个你确定不会用的武器：
池子缩小到 30 种
第 6 个武器的出现概率：每个选项 1/30 ≈ 3.3%
提升比例 ≈ 65%
```

**Banish 的性价比**：一次性的投入（消耗一个 Banish），换来整局后续的选择池纯化。

### 3.3 满级池子净化

```
武器或被动道具满级后 → 不再出现在升级选项池中
```

**数学影响**：

```
第一把武器满级后：
池子减少 1 个武器
→ 剩余池子权重从 8,130 降到 ~8,030
→ 变化极小（1.2%）

第六把武器满级后：
池子减少 6 个武器
→ 剩余池子权重从 8,130 降到 ~7,730
→ +5% 提升

但满级 6 个被动道具后（被动池从 1,370 降到 ~750）：
→ 被动道具出现概率不变（因为 6 个武器槽已满，不再出新武器）
→ 池子中只剩下未满级的被动道具
→ 每个未被 Banish 的被动道具出现概率大大提高
```

**为什么池子净化对被动道具影响巨大**：

被动道具池最初只有 1,370 权重（远小于武器的 8,130）。当 6 个被动中满级了 4-5 个：

```
原始被动池：1,370 → 出现概率 14%
满级 4 个后：~600 → 出现概率 ~7%（因为总权重池也在变）
每个未满级被动的出现概率翻了近 1 倍
```

### 3.4 满配后的保底选项

当全部武器 6 个 + 被动道具 6 个都满级且没有可进化武器时：

```
选项 1: 金币 (Gold Coin ≥ 100)
选项 2: 鸡腿 (Floor Chicken，如果血量不满)
选项 3: 鸡腿或金币的"永远选这个"选项（到达 100 次金币袋后解锁）
选项 4+: Limit Break 属性升级（如果开启）
```

**关键细节**：
- "Always Floor Chicken" 选项只有在血量不满时给鸡腿，满血时自动变金币
- 这是满配后的"安慰奖"，也是 Limit Break 系统的入口
- Limit Break 开启时，第三个选项被"升级已有武器属性"取代

### 3.5 PowerUp 系统对升级的影响

PowerUp 系统（金币永久升级）直接影响升级时的操作能力：

| PowerUp | 满级效果 | 初始价格 | 对决策的影响 |
|---------|---------|---------|------------|
| Reroll | +3 次重roll | 1,000 | 关键节点的"后悔药" |
| Skip | +3 次跳过 | 1,000 | 拒绝三个垃圾选项 |
| Banish | +3 次永久移除 | 1,000 | 池子净化工具 |
| Seal | +5 个永久封印位 | 2,000 | 全局池子净化（一劳永逸） |

**Seal vs Banish 的区别**：

```
Banish（逐局）：
- 每局可用 3 次（PowerUp 满级）
- 只能在一局内生效
- 需要花升级机会去操作

Seal（永久）：
- 在开始前的菜单设置
- 永久生效（可以改动）
- 被 Seal 的武器/被动在本局**完全不会出现**
- 推荐用在"永远不会选"的武器上
```

**最优 Seal 策略**：把一些"你不会走那个流派"的武器 Seal 掉 → 池子从一开始就小了 → 整局各种概率都变高。

## 四、Luck 属性的深层影响

### 4.1 Luck 在升级系统中的所有影响

来自 Wiki，Luck 影响升级系统的 **四个独立维度**：

| 维度 | 公式 | 影响的决策 |
|------|------|-----------|
| 拥有物品偏向 | `1 + 0.3^(x-1) × totalLuck` | 是否更容易升级已有武器 |
| 第四个选项 | `1 - 1/totalLuck` | 是否有更多选择 |
| 宝箱等级 | `levelChance × totalLuck` | 宝箱内奖励的质量 |
| 武器特效触发 | `effectChance × totalLuck` | 暴击/特效概率 |

### 4.2 第四个选项的出现概率（精确计算）

```
chanceFourth = 1 - (1 / totalLuck)
```

| 显示 Luck | totalLuck | 第四选项概率 |
|-----------|-----------|-----------|
| 0%（基础） | 100 | 0% |
| +30%（基础+PowerUp） | 130 | 23.1% |
| +50%（基础+Clover Lv.5） | 150 | 33.3% |
| +80% | 180 | 44.4% |
| +100% | 200 | 50% |
| +200% | 300 | 66.7% |
| +400% | 500 | 80% |
| +900% | 1,000 | 90% |

**设计解读**：Vanilla Luck（无加成）根本没有概率看到第四选项。至少需要 +30% Luck（PowerUp 满级3级）才能有 ~23% 概率。这是一个"高 Luck 高收益"的设计。

### 4.3 拥有物品偏向的偶/奇等级交替

从 Wiki：

```
ownedChance = 1 + 0.3^(x-1) × totalLuck
where x = 2 if the player's level is even and x = 1 if the level is odd.
```

**逐级计算**：

默认 Luck 无加成（totalLuck = 100）：

| 等级 | x | 公式 | ownedChance |
|------|---|------|-----------|
| 奇数 | 1 | `1 + 0.3^0 × 100 = 1 + 1 × 100` | 101（几乎必定偏向） |
| 偶数 | 2 | `1 + 0.3^1 × 100 = 1 + 0.3 × 100` | 31（有偏向但不绝对） |

有 +50% Luck（totalLuck = 150）：

| 等级 | x | 公式 | ownedChance |
|------|---|------|-----------|
| 奇数 | 1 | `1 + 0.3^0 × 150 = 1 + 150` | 151 |
| 偶数 | 2 | `1 + 0.3^1 × 150 = 1 + 45` | 46 |

**设计意图**：奇数级时系统强力偏向"升级已有武器"，偶数级时偏向弱化。这产生了自然的节奏：

```
奇数级（Lv.1, 3, 5, 7...）: 倾向于升级已有武器 → 武器等级快速增长
偶数级（Lv.2, 4, 6, 8...）: 更倾向于拿新武器 → 拓展武器种类
```

**实际操作**：检查是连续进行两次的。如果两次都成功，从已有武器中挑两个不同的做选项。如果第二次抽到了和第一次一样的，丢弃它然后从全武器池中抽取。

### 4.4 Luck 的增加方式汇总

| 来源 | 最大 Luck 加成 | 可实现性 |
|------|-------------|---------|
| PowerUp（金币升级） | +30% | 永恒，所有局生效 |
| Clover（被动道具） | +50%（Lv.5） | 需要占一个被动槽 |
| Little Clover（拾取品） | +10%（单次） | 地图有限个 |
| Hyper 模式 | +10%~+20% | 选 Hyper 时自动 |
| Inverse 模式 | +20% | 选 Inverse 时自动 |
| 金币蛋（Golden Egg） | 无上限，每颗 1% | 极度后期内容 |
| Keitha（角色） | 每级 1%，无上限 | 特定角色 |

### 4.5 Luck 的其他影响（非升级）

同样来自 Wiki Luck 页面：

| 场景 | 公式 | 效果 |
|------|------|------|
| 光源掉落 | `rarity × totalLuck` | 高 Luck 下光源掉落更好东西 |
| 光源生成 | `mapChance × totalLuck` | 更频繁生成光源 |
| 地图事件 | `eventChance / totalLuck` | 某些事件概率受 Luck 影响 |
| 陷阱冷却 | `trapCD × totalLuck` | 高 Luck 下陷阱失效更久 |
| 关卡道具多倍 | `1 - chance^totalLuck` | 某些地图道具可以刷出多个 |

## 五、决策树的复杂性与重量

### 5.1 决策密度分析

```
一局 ~30 分钟 ≈ 60 次升级
每次 3-4 个选项 ≈ 每局 180-240 个微决策
≈ 每 8-10 秒一个决策

每次决策的影响：
- 当前：选择 A 还是 B
- 后续：A 的选择会改变未来的选项概率（池子变化/Banish 影响）
- 全局：影响最终配装和生存能力
```

### 5.2 路径依赖深度

```
决策节点 1: Lv.2 选了 Magic Wand
→ "好的，我走冷却路线了"
→ 之后的决策偏向 Empty Tome（冷却被动）
→ 如果幸运拿到 Empty Tome → 进化 Holy Wand
→ 如果不幸直到 15 分钟也没拿到 Empty Tome → 只能走泛用路线

每个决策节点都是一个"分支"
Lv.2 → Lv.4 → Lv.6 → ... → Lv.60
每个分支要么"加固现有路线"，要么"转向另一条路线"
```

**这与传统 Roguelike（如 Slay the Spire）的不同**：

| VS | Slay the Spire |
|----|---------------|
| 每 ~20 秒一个决策 | 每 ~2 分钟一个决策 |
| 决策重量中等 | 决策重量高 |
| 可以有 1-2 个"废决策"不致命 | 1 个废决策可能直接结束游戏 |
| 容错率高（可以走泛用路线） | 容错率低（必须专注路线） |

### 5.3 池子大小 vs 决策质量的关系

```
武器池权重：8,130（全 DLC）
被动池权重：1,370（全 DLC）

没有用 Banish/Seal 的情况下：
每个选项 ~14% 概率是被动，~86% 概率是武器
"被动道具不好拿"是这个池子权重比的直接后果

使用 Seal/Banish 后：
池子缩小 → 想要的概率变大
这是"控制随机"的核心手段
```

### 5.4 Seal 系统的全局净化

Seal 是在**升级菜单外**的永久池子管理：

```
使用 Seal 封住 5 个你确定不会玩的武器
→ 前 6 个武器槽选到"有效武器"的概率从 86% → 显著提升
→ 被动道具出现的"相对概率"不变（权重比没变），
  但"绝对有效选项"增加

最佳 Seal 策略：
1. 封住不需要的武器（如某些特殊武器）
2. 被动道具不建议封（因为被动池本来就小 + 每个被动都有进化价值）
3. 封掉"前期拿到会卡槽"的武器
```

## 六、Key Takeaways

1. **Rarity 1 的武器出现概率仅 0.011%**：每次看到超稀有武器都像是在"中彩票"，这种稀缺感驱动了反复游玩的动机
2. **奇数级升级/偶数级新武器**：拥有物品偏向的偶/奇交替创造了自然的"升级→获取→升级→获取"节奏
3. **8,130 : 1,370 的权重比解释了"为什么被动道具这么难拿"**：武器和被动池的不对称设计迫使玩家必须战略性地留下被动槽位
4. **Lv.20 和 Lv.40 是两道人工门槛**：用突然飙升的经验需求和临时 Growth 爆发制造"瓶颈→突破"的节奏
5. **Banish 不是"后悔药"，是概率优化工具**：消耗一次机会，永久缩小池子——这是"主动控制随机"的设计
6. **Luck 不是单一属性，是决策链放大器**：它同时影响选项数量、拥有物品偏向、宝箱质量和武器特效——覆盖面极广
