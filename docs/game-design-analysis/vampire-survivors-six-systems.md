# Vampire Survivors — 支撑底层六大系统深度分析

> **关联文档**：[极简操作 + 自动攻击深度分析](./vampire-survivors-minimal-controls)
> **分析聚焦**："玩家只需要移动"这个体验是靠哪些底层系统支撑的

---

## 概述

Vampire Survivors 的"极简操作 + 自动攻击"不是简单地把攻击做成交互缺失，而是由六大底层系统共同支撑起来的精密设计。每一个系统都承担了原本属于玩家的操作负担，把它们自动化，同时保留策略深度。

本文将用来自 Wiki 的精确数据，逐一拆解这六大系统的实现逻辑、数值设计和设计意图。

---

## 一、武器冷却系统（Cooldown）

### 1.1 冷却的数值基础

```
基础冷却 = 100%（默认）
冷却下限 = 10%（再快也快不过 10%）
```

冷却不是"频率"，而是"延迟之间的倍率"。100% 冷却 ≈ 武器按照设计间隔发射；50% 冷却 ≈ 一半的间隔 → 发射频率翻倍。

### 1.2 冷却缩减的叠加方式

```
最终冷却 = 基础冷却 - 所有冷却缩减的总和（加法叠加）

例：
基础冷却 = 100%
Empty Tome Lv.5 = -40%
Cooldown PowerUp = -5%
角色加成 = -10%
Arcana (Silent Old Sanctuary, 5空槽) = -40%
────────────────────────────
最终冷却 = 100% - 95% = 5% → 但下限是 10%
→ 所以实际效果 = 10%
```

**加法叠加意味着什么**：
- 多个冷却缩减来源叠加收益极高（边际效应递增）
- 从 100% 减到 50% → 发射频率翻倍
- 从 50% 减到 10% → 发射频率再翻 5 倍
- 这意味着冷却配置的收益率不是线性，而是**指数级**的

### 1.3 冷却缩减的来源

| 来源 | 最大效果 | 获取方式 |
|------|---------|---------|
| Empty Tome（被动道具） | -40% | 升级时选择，可进化 |
| Cooldown PowerUp（永久升级） | -5% | 金币升级，仅需 2 级 |
| 角色初始加成 | -10% ~ -30% | 角色选择 |
| Silent Old Sanctuary (XX) | -8%/空武器槽，最多 -40% | Arcana 选择 |
| Tragic Princess (III) | 移动时减少冷却（无下限） | Arcana 选择 |

### 1.4 升级重置冷却

当玩家在升级中选择一个已有武器时，该武器立即重置冷却并发射一次：

```
玩家升级 → 选择已有武器
→ 武器冷却重置为 0
→ 武器立即触发一次攻击
→ 然后从头开始冷却循环
```

**设计意义**：
- 升级本身成为了一个"输出爆发"事件
- 对慢冷却武器尤其有价值（如圣水，原本冷却 10 秒，升级后立刻砸一瓶）
- 这给了玩家一个隐性策略：**锁定慢冷却武器不升满，每次升级来重置冷却**（不过满级后池子净化也有价值）

### 1.5 冷却对不同武器的影响差异

```
短冷却武器（如小刀：0.3-0.5 秒）：
- 冷却缩减让它们变成"机关枪"
- 每秒触发几十次

长冷却武器（如圣水：10 秒）：
- 冷却缩减让它们从"每 10 秒扔一瓶"到"每 3 秒扔三瓶"
- 性质完全改变
```

冷却系统的设计平衡：短冷却武器靠频率，长冷却武器靠单次伤害——冷却缩减对两种武器都有收益，但收益形式完全不同。

## 二、自动瞄准系统（Auto-Aim）

### 2.1 目标选择逻辑

虽然 Wiki 没有给出确切的瞄准算法源码，但从武器行为可以反推出目标选择的逻辑：

```
for each weapon fire:
    if weapon.type == "方向投射":
        目标 = 最近的敌人
        投射物朝目标方向发射
    
    if weapon.type == "环绕":
        投射物围绕玩家旋转
        碰撞到任何敌人即触发
    
    if weapon.type == "追踪":
        目标 = 最近的敌人
        投射物持续追踪目标移动
    
    if weapon.type == "全向扩散":
        朝 360° 方向发射
        覆盖范围内的所有方向
    
    if weapon.type == "地面放置":
        在玩家周围/面前放置
        敌人走进范围触发
```

### 2.2 武器弹道模式分类（完整版，含 DLC）

从 Wiki 的 Weapons 页面可以归纳出 12 种基本弹道模式：

| 编号 | 模式 | 代表武器 | 攻击覆盖 |
|------|------|---------|---------|
| 1 | 定向扇形 | Whip, Bloody Tear | 前方 60° 扇形 |
| 2 | 定向直线 | Knife, Thousand Edge | 前方直线 |
| 3 | 全向 360° | Axe, Death Spiral | 全方向 |
| 4 | 环绕轨道 | King Bible, Unholy Vespers | 近身全向 |
| 5 | 追踪 | Magic Wand, Holy Wand | 自动追踪敌人 |
| 6 | 反弹 | Runetracer, NO FUTURE | 弹射墙壁，持续飞行 |
| 7 | 地面部署 | Santa Water, La Borra | 地面区域 |
| 8 | 空中打击 | Lightning Ring, Thunder Loop | 全屏随机 |
| 9 | 召唤物 | Gatti Amari, Vicious Hunger | 自主 AI 单位 |
| 10 | 爆炸 | Cherry Bomb | 撞击后爆炸 |
| 11 | 光束 | Victory Sword | 持续光束 |
| 12 | 特殊 | Pentagram, Gorgeous Moon | 全屏清除 |

### 2.3 自动瞄准的战术影响

因为瞄准是自动的，玩家对"攻击方向"的控制权仅限于角色的朝向。

```
玩家朝向 ← → 面朝右 → 鞭子向右打
            面朝左 → 鞭子向左打
```

对于方向投射武器（鞭子、小刀），玩家可以通过移动来间接控制攻击方向：

- 面朝右走 → 鞭子打右边
- 面朝左走 → 鞭子打左边
- 按左走但同时面朝右？不行，VS 的人物面向和移动方向绑定

**设计意义**：面朝 = 移动方向，意味着玩家通过"你往哪里走"就决定了"你往哪里打"——移动和攻击方向深度绑定，进一步强化了"唯一操作是移动"的设计。

### 2.4 暴击系统（Critical Hit）与自动瞄准的联动

```
暴击判定：自动计算
暴击率：基础值 + Luck 修正 + 武器/角色加成
暴击效果：伤害 × 2（默认）
```

暴击不需要玩家任何操作——自动瞄准 + 自动暴击判定，伤害计算完全自动化，玩家只需要在选武器时考虑"这个武器暴击收益高不高"。

## 三、经验宝石系统（Experience Gem）

### 3.1 经验宝石的种类与数值

| 颜色 | 经验值 | 说明 |
|------|--------|------|
| 蓝色 | 1-2 XP | 最小的经验块 |
| 绿色 | 5-9 XP | 中等经验 |
| 红色 | 10+ XP | 大经验块 |
| 特殊（Gorgeous Moon） | 1 XP（但数量极多） | 橙色/黄色/品红色 |

### 3.2 经验宝石的生成逻辑

```
击杀敌人 → 掉落 1 个经验宝石
            宝石类型 = f(敌人类型, 幸运值, 地图)
            
累计 400 个地面经验宝石 → 所有新经验自动合并到大红宝石
→ 防止过多屏幕粒子拖慢性能
```

**400 个上限设计**：这是性能保护机制。如果玩家够强，每秒杀几十个敌人，5 秒就能产生 400+ 宝石。合并成大红宝石既避免性能问题，也创建了视觉反馈——"你杀了足够多了，现在合并成一个大的"。

### 3.3 Growth 属性对经验的影响

```
实际获得经验 = 宝石基础值 × (1 + Growth%)
```

Growth 属性不是增加掉落率，而是**放大经验宝石的价值**。

| Growth 来源 | 数值 |
|------------|------|
| 基础 | 100%（1×） |
| Crown（王冠）被动道具 | 每级 +20%，满级 +100% |
| Wicked Season (XIII) Arcana | Growth +100% 持续 30 秒（周期性触发） |
| 角色加成 | 0% ~ +50% |
| Hurry Mode | +25% 经验获取 |

**设计意义**：
- Growth 是加速成长的核心属性，但需要牺牲一个被动槽位
- Crown 没有对应的进化武器（它本身就是进化催化剂），所以持有没有"双重价值"
- 选择 Growth 意味着"我要快速成型"，但代价是占了一个被动槽

### 3.4 经验曲线与等级门槛

从 Wiki 的精确数据：

```
Lv.1→2:  需要 5 XP (累计 5)
Lv.2→3:  需要 15 XP (累计 20)      每级 +10
Lv.9→10: 需要 85 XP (累计 405)
Lv.19→20:需要 185 XP (累计 1,805)

[等级门槛]
Lv.20→21:需要 600 XP (累计 2,600)   ← +100% Growth（突破后有加速）
Lv.21→22:需要 266.5 XP (累计 2,866.5) ← 每级 +13

Lv.39→40:需要 498.5 XP (累计 9,886.5)

[等级门槛]
Lv.40→41:需要 2,400 XP (累计 12,800)  ← +100% Growth（突破后有加速）
Lv.41→42:需要 648 XP (累计 13,448)    ← 每级 +16
```

**关键洞察**：Lv.20 和 Lv.40 是两道"人工门槛"，突然需要远超正常等级所需的经验，但同时给 +100% Growth 作为补偿。

**设计意图**：
1. 制造两个"成长瓶颈"，迫使玩家在中期面对经验不足的困境
2. 瓶颈后的 Growth 爆发让玩家快速冲过下一阶段
3. 这种"瓶颈 → 突破 → 加速"的节奏感和武器进化是同步的

### 3.5 经验宝石自动拾取的心理学

经验宝石自动进入 Magnet 范围 → 玩家不需要按键拾取。但这个设计有更深的心理学机制：

```
宝石掉落 → 玩家看到宝石 → 本能地朝宝石移动
          → 走入 Magnet 范围 → 宝石自动飞向玩家
          → 获得经验 → 积累到升级 → 暂停选择
          → 变强
```

这个链的价值在于：

- **宝石是天然的"路径引导"**：玩家看到宝石堆就会不自觉往那边走，这给了玩家无意识的微观目标
- **靠近宝石 = 变强**：每次拾取宝石都是一个小反馈
- **自动拾取消除了打断**：不需要在战斗中停下来捡东西
- **Magnet 范围可以扩大**：Attractorb 升级 → 更大范围 → 更轻松的拾取体验

## 四、升级暂停保护帧系统（Level-Up Pause & Invulnerability）

### 4.1 升级时的完整事件序列

```
[1] 经验条满 → 升级触发
[2] 屏幕出现粒子动画（经验宝石从屏幕上方掉落）
[3] 游戏完全暂停（所有敌人、弹道冻结）
[4] 弹出 3 或 4 个选项
[5] 玩家选择（或 Skip/Reroll/Banish）
[6] 游戏恢复
[7] 角色获得短暂无敌帧
[8] 如果选择的是已有武器：立即重置冷却 + 触发一次攻击
```

### 4.2 暂停的设计价值

**为什么升级要暂停？**

VS 的节奏：玩家在极速走位躲怪，如果不在升级时暂停，玩家要么：
- A：选技能时被怪打死
- B：为了安全边走边选 → 失误率高
- C：干脆不升级 → 放弃成长

三种情况都会严重破坏体验。**暂停让"选技能"成为完全安全的决策空间**。

### 4.3 无敌帧的数值

Wiki 明确了升级后的无敌帧，但没有给出精确的持续时间。从社区经验来看：

```
无敌帧持续时间 ≈ 0.5-1.0 秒
作用：防止"刚选完就被瞬移过来的怪打死"
```

**设计意义**：无敌帧解决了升级暂停恢复后的"瞬间死亡"问题。如果没有这个无敌帧，玩家可能会经历：

```
恢复 → 敌人正好在脚下 → 被秒 → 好不容易攒的升级废了 → 挫败感
```

这个 0.5-1 秒的无敌窗口给了玩家"重新适应战场"的时间。

### 4.4 冷却重置作为"战斗力恢复"

升级时的一个隐藏机制：如果选的是已有武器（升级），该武器**立即重置冷却并发射一次**。

```
升级前：武器 A 冷却还有 3 秒
升级：选择武器 A → 冷却重置为 0 → 武器 A 立即开火一次
升级后：武器 A 冷却从 0 开始计算
```

这个机制确保了玩家在升级暂停恢复后不是"零输出"状态，而是"满输出"状态。

### 4.5 四个选项的 Luck 阈值

```
默认：3 个选项
第四个选项概率：chanceFourth = 1 - (1 / totalLuck)

例：
totalLuck = 100% (1×)  → 0% 概率有第四选项
totalLuck = 200% (2×)  → 50% 概率有第四选项
totalLuck = 300% (3×)  → 66.7% 概率有第四选项
totalLuck = 500% (5×)  → 80% 概率有第四选项
totalLuck = 1000% (10×) → 90% 概率有第四选项
```

更多选项 = 更高自由度 = 更容易找到想要的武器/被动。这是 Luck 最有价值的应用之一。

## 五、击退系统（Knockback）

### 5.1 击退的基本逻辑

```
命中判定成功 → 判断武器是否有击退属性
→ 计算击退距离（基础值 × 击退倍率）
→ 沿"玩家→敌人"方向施加力
→ 敌人被推开一定距离
```

### 5.2 各武器的击退效果

| 武器 | 击退力 | 说明 |
|------|--------|------|
| Whip / Bloody Tear | 中 | 标准击退 |
| King Bible / Unholy Vespers | 强 | 环绕击退，创造安全圈 |
| Garlic / Soul Eater | 极弱（持续伤害） | 主要靠伤害，击退是附带的 |
| Knife / Thousand Edge | 弱 | 高频低击退 |
| Axe / Death Spiral | 中 | 全向击退 |
| Santa Water / La Borra | 无（地面效果） | 不击退，纯伤害 |
| Lightning Ring / Thunder Loop | 无 | 空中打击，不击退 |

### 5.3 击退的战术价值

击退是"自动攻击"系统的关键辅助系统。分析它的战术作用：

```
没有击退的情况：
敌人持续向玩家移动
玩家必须不断调整走位
容错率低（稍微停一步就被包围）

有击退的情况：
武器自动推开靠近的敌人
在玩家周围维持一个"安全圈"
玩家可以短暂停顿喘气
容错率高
```

**击退创造的操作缓冲区**：即使玩家走位失误，击退系统也会推开敌人，给玩家时间调整位置。

### 5.4 击退与走位的协作

击退不是代替走位，而是和走位协同工作：

```
玩家往右走 → 击退方向是"从玩家朝外" → 右边敌人被击退 + 玩家向右移动
→ 效果：玩家向左边的敌人靠近 → 左边被击退

结论：击退方向与玩家移动方向配合
→ 玩家面对的方向 = 攻击和击退的主要方向
→ 玩家背对的方向 = 靠近敌人的方向
```

这创造了一个有趣的走位模式：**永远不要停在一个方向上**，因为只有移动中才能控制击退方向。

### 5.5 击退的 Limitation

击退也有局限：

```
- 击退对 Boss 无效（或效果极低）
- 击退对大规模敌人群效果减弱（前面的被推开，后面的补上）
- 击退距离有限（推不开到安全距离就会重新围上来）
- 高难度下（Inverse Mode），敌人移速增加 → 击退的效果减弱
```

所以击退是辅助系统，不是万能的。它是"让自动攻击可行"的关键组件，但不是让游戏变简单的核心。

### 5.6 Mask of the Red Death 的极端案例

Red Death 角色移速 +100%。高速移动 + 击退系统 = 直接冲过敌人群。这个角色证明了移速和击退的联动可以强大到什么程度。

## 六、Magnet 自动拾取系统

### 6.1 Magnet 的数值基础

```
Magnet 基础范围：约屏幕的 1/4 ~ 1/3 半径
Attractorb（吸引力石）：每级 +25%，满级 +125%

Lv.1: 125% 范围
Lv.2: 150% 范围
Lv.3: 175% 范围
Lv.4: 200% 范围
Lv.5: 225% 范围（最大范围）
```

### 6.2 Magnet 拾取的物品类型

| 物品 | Magnet 自动拾取？ | 备注 |
|------|-----------------|------|
| 经验宝石 | ✅ 自动拾取 | 进入范围自动吸附 |
| 金币 | ✅ 自动拾取 | 同宝石 |
| 金币袋 | ✅ 自动拾取 | 各种面额 |
| 鸡腿 (Floor Chicken) | ✅ 自动拾取 | 必须在范围内 |
| 小道具 (Clover, Rerollo 等) | ✅ 自动拾取 | 范围触发 |
| **宝箱** | ❌ **必须走近触碰** | 唯一例外 |

### 6.3 Magnet 不作用于宝箱的设计意图

这个对比很关键：

```
经验宝石 → Magnet 自动拾取 → 无风险、无操作 → 常规反馈
宝箱 → 必须手动走近 → 有风险、有决策 → 重要事件

经验宝石 = 填充物（丰富但安全）
宝箱 = 奖杯（稀有但需要主动获取）
```

如果宝箱也能被 Magnet 吸过来，那么：
- 玩家不需要做出"冒不冒险去拿宝箱"的决策 → 少了一层策略
- 宝箱的仪式感降低 → 变成又一个自动拾取
- 进化时机不再可控 → 玩家不需要规划"什么时候去拿进化"

### 6.4 Magnet 与走位的关系

Magnet 的反直觉效果：**越大越好，但也不是无脑大**。

```
小 Magnet：
- 需要精确走位到宝石旁边 → 更专注的路径规划
- 但容易错过边角的宝石 → 成长效率低

大 Magnet (Attractorb Lv.5)：
- 大范围自动吸附 → 几乎不丢经验
- 但：引诱玩家去走更冒险的路线（因为范围大，想去吃远方的怪）

中等 Magnet (Attractorb Lv.2-3)：
- 多数情况下是最优选择
- 覆盖主要区域但不至于过度激进
```

### 6.5 Magnet 与 Vicious Hunger（猫武器）的联动

猫武器（Gatti Amari / Vicious Hunger）有个著名的彩蛋行为：当玩家开宝箱时，升级选项中有猫武器时，猫会代替经验宝石从屏幕上方掉落。这与 Magnet 无关，但说明了一个有趣的点——Magnet 只处理"地面对象"，而"从屏幕上方降落的猫"是动画事件，不受 Magnet 影响。

### 6.6 Magnet 的隐藏功能：信息反馈

Magnet 不仅是功能系统，它也是信息反馈系统：
- 当宝石飞向玩家时 → 玩家知道"我在捡经验"
- 当宝石停在远处不动时 → 玩家知道"我离远了"
- 扩大的 Magnet 范围 → 玩家能感受到"我的道具生效了"的反馈

**设计意义**：Magnet 的自动拾取不是静默的，而是带着视觉反馈的——宝石飞向角色的轨迹本身就是"你在变强"的提示。

## 七、六大系统的协同工作

### 7.1 系统依赖关系

```
自动瞄准  ──→  击退
   │                 │
   │                 ↓
   ├─→  武器冷却  ──→   输出效率
   │
   ↓
经验宝石 ←── Magnet
   │
   ↓
升级暂停 ←── 无敌帧
```

### 7.2 系统链：从"玩家移动"到"变强"

```
玩家移动
→ 控制面朝方向 → 影响自动瞄准的攻击方向
→ 靠近敌人 → 触发自动攻击 → 造成伤害
→ 击退推开附近敌人 → 创造安全空间
→ 击杀敌人 → 经验宝石掉落
→ Magnet 自动吸附宝石 → 经验增长
→ 升级 → 暂停 → 选择 → 冷却重置 + 无敌帧
→ 回到"移动"（更强了）
```

这条链的核心是：**玩家的唯一输入（移动）通过六大系统转化为所有其他游戏行为**。

### 7.3 哪个系统最重要？

```
没有武器冷却 → 武器不发射 → 玩家死了
没有自动瞄准 → 需要手瞄 → 操作复杂度暴增
没有经验宝石 → 无法升级 → 永远 1 级
没有升级暂停 → 选技能时被打死 → 挫败感
没有击退 → 走位压力极大 → 容错率为零
没有 Magnet → 需要手捡宝石 → 操作打断

结论：去掉任何一个系统，"极简操作"都会失效
这六个系统缺一不可
```

## 八、Key Takeaways

1. **冷却系统是节奏控制的核心**：加法叠加让冷却缩减的收益率递增，冷却缩减从 50% 到 10% 带来的频率提升远大于 100% 到 50%
2. **自动瞄准不简单**：12 种弹道模式覆盖不同的战术场景，玩家通过选择武器种类来"编程"自动攻击的行为
3. **经验宝石是微观路径引导**：自动拾取消除了操作打断，400 颗合并上限是性能和视觉的双重优化
4. **升级暂停保护决策空间**：完全暂停让选技能无压力，无敌帧防止"刚选完就死"的挫败
5. **击退创造了操作缓冲区**：不是代替走位，而是给走位更多容错空间；与移动方向绑定产生"拉扯"技巧
6. **Magnet ≠ 自动拾取一切**：宝箱的主动获取保留了"冒险 vs 收益"的决策层
