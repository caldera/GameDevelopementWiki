# Vampire Survivors — Roguelite 升级决策树 · Key Takeaways 深度分析

### 6.1 🎰 Rarity 1 — 超稀缺驱动的反复游玩动机

**一句话**：Rarity 1 的武器出现概率仅 0.011%，每次看到都像中彩票——稀缺感本身就是重玩驱动力。

#### 6.1.1 数学层面

在 8,130 的武器总权重中，Rarity 1 意味着：

```
每次升级看到它的概率 = 1 / 8,130 ≈ 0.0123%

一局 60 次升级中，至少看到一次的概率 ≈ 1 - (1 - 0.000123)^60 ≈ 0.74%

也就是说：平均需要 135 局才能看到一次 Rarity 1 武器

即使看到了，你还要面对 3-4 个选项中的竞争
→ 最终选到它的概率更低
```

#### 6.1.2 心理学层面

这是**可变奖励（Variable Reward）** 在游戏设计中的经典应用——与老虎机同源：

```
固定奖励：每次升级都能拿到某物 → 习以为常，没有惊喜
超低概率奖励：几乎不可能出现 → 但当它出现时，兴奋感极强

这种"几乎不可能但万一呢"的期待感让玩家愿意反复开局
每次升级都抱着一丝"这次会不会出那个武器"的幻想
```

#### 6.1.3 游戏中的 Rarity 1 武器

| 武器 | Rarity | 解锁方式 | 效果概述 |
|------|--------|---------|---------|
| Victory Sword | 1 | 收集 30 个遗物 | 超大范围剑气斩，极高伤害 |
| Sole Solution | 1 | Victory Sword 进化 | 全屏清除 + 持续伤害 |
| (部分 DLC 隐藏武器) | 1 | 特定条件 | 各有独特机制 |

#### 6.1.4 设计启示

- **超低概率创造记忆峰值**：一局 60 次升级中，玩家不会记得 59 次普通选择，但会永远记得"那次 Rarity 1 武器出现了"
- **稀缺≠强度**：Rarity 1 武器并不一定比 Rarity 100 的武器强很多——稀缺本身是游戏体验的一部分
- **追求驱动**：知道"这个东西几乎不会出现"反而让玩家更想去追求它，因为一旦拿到就是独一无二的一局

### 6.2 🔄 奇数级升级 / 偶数级新武器 — 偶奇交替的隐性节奏

**一句话**：拥有物品偏向的偶/奇交替创造了自然的"升级→获取→升级→获取"节奏。

#### 6.2.1 公式回顾

从 Wiki：

```
ownedChance = 1 + 0.3^(x-1) × totalLuck

x = 2 偶数级，x = 1 奇数级
```

默认 Luck（totalLuck = 100）：

| 等级 | ownedChance | 偏向强度 |
|------|-----------|---------|
| 奇数 | 101 | 极高（'几乎必定'偏向已有武器） |
| 偶数 | 31 | 中等（有偏向但不绝对） |

#### 6.2.2 这个设计解决了什么问题

没有偶/奇交替，只用"随机"的话，玩家可能会经历：

```
连续的奇数级：
Lv.1 拿了新武器 → Lv.3 又拿了新武器 → Lv.5 又拿了新武器
→ 6 个槽很快填满，但所有武器都是 1 级
→ 输出不足 → 站不住
```

有了偶/奇交替：

```
Lv.1（奇数）：极大概率升级已有武器 → 首个武器升到 2 级
Lv.2（偶数）：偏向稍弱 → 50%概率拿新武器、50%升级已有的
Lv.3（奇数）：极大概率升级已有的 → 继续强化现有武器
```

**实际效果**：玩家被隐性地引导去"先把手上武器养起来"，而不是每级都拿新武器。

#### 6.2.3 操作层面的感知

玩家几乎**不会感知到**这个偏向系统的存在。它们不会看到公式，只知道"好像经常抽到已经有的武器"。

这正是好设计的特点——规则在幕后起作用，玩家感受到的是"游戏节奏很自然"，而不是看到一堆数学公式。

#### 6.2.4 设计启示

- **规则越隐性越好**：玩家不需要理解公式，只要感受到"节奏对了"
- **偶/奇交替比纯随机更好**：消除了"连续拿新武器"的极端情况
- **对 Luck 的依赖恰到好处**：初代玩家也能体验到平均节奏，高 Luck 玩家体验到的"专注"感更强

### 6.3 ⚖️ 8,130 : 1,370 — 池子权重比塑造的被动道具稀缺

**一句话**：武器和被动池的不对称设计（86%:14%）迫使玩家必须战略性地留下被动槽位。

#### 6.3.1 池子权重比的后果

```
武器池总权重：8,130
被动池总权重：1,370
武器：被动 ≈ 86 : 14

每次升级选项：
平均 3 × 0.86 ≈ 2.58 个是武器
平均 3 × 0.14 ≈ 0.42 个是被动
```

这意味着：**每 3 次升级才能看到 1 次被动道具选项**。

```
要在 6 个被动槽全部满级（6 × 5 = 30 级）：
→ 需要被动道具出现至少 30 次（加上首次获取和升级）
→ 如果每 3 次升级出现 1 次被动 → 需要 ~90 次升级
→ 一局才 60 次升级左右

→ 这意味着：单靠自然升级很难把 6 个被动全部升满
→ 宝箱提供额外升级 → 这是为什么宝箱对被动道具很重要
```

#### 6.3.2 被动道具先拿还是后拿的策略

```
早点拿被动（如 5 分钟前就拿到 3 个被动）：
- 好处：可以更早注意到进化催化剂、被动等级慢慢升
- 坏处：武器没成型前就占槽位，可能错过关键武器

晚点拿被动（优先装满 6 个武器槽）：
- 好处：武器成型快、前期输出足
- 坏处：要凑进化时发现重要的被动还没拿、或者拿到了但等级不够
```

#### 6.3.3 设计者的意图

8,130 : 1,370 的比值不是随意的。如果你去掉所有 DLC，只算基础游戏：

```
基础游戏武器池：较小的数值
基础游戏被动池：也较小，但比值相近

→ 这个比值是故意设计的，不是 DLC 膨胀的结果
→ 设计者想要"被动道具比武器难拿"的体验
```

**为什么**：
- 让"拿到合适被动"成为一种满足感
- 鼓励战略性选择（不是无脑拿所有被动）
- 制造"就差这个被动就能进化了"的紧张感

#### 6.3.4 设计启示

- **不对称池子比平均池子更有趣**：如果武器和被动 50:50，游戏会变得太平淡
- **人为制造稀缺**：被动道具少但重要，所以每次看到"核心被动"都很有满足感
- **宝箱成为被动升级的关键**：因为自然升级不够 → 宝箱提供的额外升级对被动道具影响更大

### 6.4 🧱 Lv.20 和 Lv.40 — 两道人工门槛的节奏设计

**一句话**：用突然飙升的经验需求和临时 Growth 爆发制造"瓶颈→突破→加速"的节奏。

#### 6.4.1 门槛的数学对比

```
正常升级段（Lv.2-20）：每级 +10 XP
Lv.20→21：需要 +600 XP（比正常高 60 倍）

正常升级段（Lv.21-40）：每级 +13 XP
Lv.40→41：需要 +2,400 XP（比正常高 185 倍）
```

**但别忘了 Growth 翻倍**：

```
Lv.20→21：
表面需要 600 XP，但因为 +100% Growth → 每个宝石经验翻倍
实际击杀量 ≈ 300 XP 相当

Lv.40→41：
表面需要 2,400 XP，同样 Growth 翻倍 → 实际击杀量 ≈ 1,200 XP 相当
```

#### 6.4.2 门槛时间在游戏时间轴中的位置

```
Lv.20 → 约游戏内 8-10 分钟
→ 对应第一次进化开启的时间（宝箱开始掉落进化）
→ 这是一个好时机——敌人开始增多，但你已经可以进化了

Lv.40 → 约游戏内 16-18 分钟
→ 对应第二次进化高峰（大部分武器进化后的继续成长）
→ 这还是 Limit Break 可能触及的门槛
```

**两个门槛恰好落在游戏的两个关键转折点**：

```
0-5 min   轻松发育期        → 快速升到 Lv.15
5-10 min  压力上升期        → Lv.15-20，遇到第一个门槛
10-15 min 第一次进化高峰     → Lv.20-35（突破后 Growth 加速）
15-20 min 二次进化期         → Lv.35-40，遇到第二个门槛
20-25 min 满配极限期         → Lv.40-50（再次突破加速）
25-30 min 死亡钟声           → Lv.50-60 终局
```

#### 6.4.3 设计启示

- **人工门槛 ≠ 坏设计**：门槛让玩家感受到"突破"的成就感
- **Growth 翻倍是"温柔的手"**：门槛虽然突然变高，但 Growth 翻倍让实际体验没那么痛苦
- **门槛时间点与游戏事件对齐**：不是随机放置，而是精确对应进化/满配的关键时刻

### 6.5 🗑️ Banish — 从"后悔药"到概率优化工具

**一句话**：消耗一次机会，永久缩小池子——这是"主动控制随机"的设计。

#### 6.5.1 Banish 的数学效应

```
初始武器池：50 种武器 → 每个选项出现特定武器的概率：1/50 = 2%

Banished 5 个武器后：45 种
→ 1/45 ≈ 2.22%（+11% 提升）

Banished 10 个武器后：40 种
→ 1/40 ≈ 2.50%（+25% 提升）

Banished 20 个武器后：30 种
→ 1/30 ≈ 3.33%（+67% 提升）
```

#### 6.5.2 Banish vs Reroll vs Skip 的决策重量

| 操作 | 消耗 | 收益 | 适合场景 |
|------|------|------|---------|
| Skip | 1 次 | 放弃当前，保留经验等下次 | 三个都不需要 |
| Reroll | 1 次 | 重新给三个选项 | 还差一个关键组件 |
| **Banish** | 1 次 | **永久移除一个选项**，后续全局受益 | 确定不用某武器 |

**关键是"后续全局受益"**：Banish 不是解决当前问题（选什么），而是优化未来的问题（以后出现什么）。

#### 6.5.3 Seal 与 Banish 的区别：临时 vs 永久

```
Banish（逐局，3 次）：
- 一局生效
- 消耗升级时的操作
- 灵活性高（可以根据本局需求调整）

Seal（永久，5 个位置）：
- 开始菜单设置，全局生效
- 所有武器一开始就不在池中
- 适合"永远不选"的武器

最佳实践：
Seal 掉 5 个"你完全不玩的流派"的武器
每局用 Banish 动态调整剩下的池子
```

#### 6.5.4 设计启示

- **主动控制随机 = 更高满意度**：Banish 让玩家感觉自己可以"管理"升级系统，不只是被动接受
- **池子净化是反直觉的聪明**：大多数人不会想到"扔掉不要的选项"来"提高想要的出现概率"
- **有限次数的 Banish 更有价值**：如果是无限 Banish，玩家会封掉所有不要的，池子太干净反而无聊

### 6.6 🍀 Luck — 决策链放大器，不是单一属性

**一句话**：Luck 同时影响选项数量、拥有物品偏向、宝箱质量和武器特效——覆盖面极广，是投入产出比最高的属性之一。

#### 6.6.1 Luck 影响链全景图

```
Luck 影响：
├── 升级系统
│   ├── 选项数量（第四选项概率）
│   ├── 拥有物品偏向（偶/奇交替公式）
│   └── 限制：不影响哪些选项出现（那是 Rarity 的工作）
├── 宝箱系统
│   ├── 宝箱等级（1/3/5 次的概率）
│   └── 宝箱是否触发进化
├── 武器特效
│   ├── 暴击率
│   ├── Pentagram 删物概率
│   └── 其他所有概率性武器效果
├── 光源系统
│   ├── 光源生成频率
│   └── 光源掉落质量
└── 地图事件
    ├── 事件触发概率
    └── 陷阱冷却延长
```

#### 6.6.2 为什么 Luck 是"隐藏 MVP"属性

```
在表面上：Luck 看起来只影响"暴击率"和"掉宝率"
⇩
在实际上：Luck 影响升级决策、宝箱质量、武器特效、地图事件
⇩
意味着一件事：高 Luck = 整个游戏所有概率事件都偏向你
```

对比其他属性：

| 属性 | 影响面 | 感知强度 |
|------|--------|---------|
| Might | 伤害（1 个维度） | 高（数字变化明显） |
| Cooldown | 攻击频率（1 个维度） | 高 |
| Area | 攻击范围（1 个维度） | 中等 |
| **Luck** | **10+ 个维度（升级/宝箱/武器/地图/光源...）** | **低（隐性）** |

Luck 的**隐性**既是优点也是缺点：
- 优点：高手玩家会发现"Luck 高的局什么都顺"，产生深度理解→技巧的感觉
- 缺点：新手玩家可能完全忽略 Luck，不知道它为什么重要

#### 6.6.3 最高 Luck 可能性

```
基础：100%
+ PowerUp（3 级）：+30%
+ Clover（Lv.5）：+50%
+ Hyper 模式（Il Molise）：+20%
+ Inverse 模式：+20%
+ Keitha 角色：每级 +1%（Lv.60 = +60%）
+ Wicked Season Arcana：周期性翻倍
────────────────────────
理论上 >+200%（totalLuck >300）
```

在 totalLuck = 300 时：
- 第四选项概率：`1 - 1/300 ≈ 99.7%`（几乎每次都有 4 个选项）
- 拥有物品偏向（奇数级）：`1 + 300 = 301`（几乎必定）
- 宝箱高级概率：3% × 300 = 9%（翻 3 倍）

#### 6.6.4 设计启示

- **隐性多维度属性 > 显性单维度属性**：Luck 影响多个系统，但不起眼——这创造了"只可意会"的深度
- **Luck 是跨系统粘合剂**：它将升级、宝箱、战斗、地图四个系统以概率的方式串联起来，让玩家感觉"运气越好，一切越顺"
- **概率放大器 ≠ 必得**：即使 300% Luck，选项仍然是"高概率"不是"必得"——保留了随机性的激动感
